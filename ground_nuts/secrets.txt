Lab 4: Secret Leak & Recovery

Simulate secret leak:

echo "AWS_SECRET_KEY=12345" > secret.txt
git add secret.txt
git commit -m "added secret key"
git push origin main


Oops! Now fix it:

Remove from repo:

git rm secret.txt
git commit -m "remove secret"
git push


For history cleanup (optional advanced):

git filter-branch --force --index-filter \
  'git rm --cached --ignore-unmatch secret.txt' \
  --prune-empty --tag-name-filter cat -- --all
git push origin --force --all


Rotate the secret in AWS IAM.

✅ Learning: Security incident handling with Git + AWS.

Lab 5: Merge Conflict in GitFlow

Create repo with main branch.

Create a release branch:

git checkout -b release-1.0


Add a line:

echo "Release config" >> config.txt
git commit -am "release config"


Switch to main, add a different line to the same file:

echo "Main config" >> config.txt
git commit -am "main config"


Merge release → conflict!

git merge release-1.0


Resolve manually, commit, push.

✅ Learning: Handling Git conflicts like in real CI/CD pipelines.
